## Context

README files need updating to reflect recent feature changes: new config path, alphanumeric IDs, and remove-all feature.

## Goals / Non-Goals

**Goals:**
- Update README.md with new features and file locations
- Update README-HK.md with same changes in Cantonese
- Keep documentation concise and accurate

**Non-Goals:**
- Rewriting entire README
- Adding new sections beyond documenting recent changes

## Decisions

**Decision 1: Update existing README sections rather than rewrite**
- Rationale: Preserve existing structure, only add/change what's needed
- Alternatives considered:
  - Full rewrite: More work, risk of losing useful content

**Decision 2: Keep README-HK.md in sync**
- Rationale: Both files should document same features
- Alternatives considered:
  - Only update README.md: Users reading HK version miss updates

## Risks / Trade-offs

- **Risk**: Documentation might become outdated again
  - **Mitigation**: This change brings docs up to date; future changes should update docs too
